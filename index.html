porter : private package manager

A self-hosted package manager built for team collaboration.

Easily search, install and publish applications from the comfort of your internal network. Porter is written in bash, works on OSX and Linux and has zero dependencies.

Install via Homebrew: brew install porter
View source: http://github.com/ricardobeat/porter

search 
porter search [string]

Search the remote repo for packages matching [string]. A full-text search on package files is performed.

install 
porter install [package-name]

Fetch and install [package-name] from the remote repo. Packages are a bundle containing a porter.sh file. The package defines preinstall steps and which binaries will be linked into /usr/local/bin.

uninstall 
porter uninstall [package-name]

Remove symlinks from /usr/local/bin. Any other modifications made to the system must be reversed by the package's own uninstall hooks.

list 
porter uninstall [package-name]

Remove symlinks from /usr/local/bin, after running any uninstall hooks.

link 
porter link [package-name]

Link binaries using the porter.sh file in the current folder. Useful for development.

publish 
porter publish [package-name]

Package and publish [package-name]. The folder will be zipped and uploaded to the remote defined in your .porter-config over SSH.
